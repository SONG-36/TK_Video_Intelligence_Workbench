# Working Area

This directory is for temporary planning, audits, and iteration materials.

Working files are not formal authority sources.

Retained file:

- `CURRENT_IMPLEMENTATION_AUDIT.md` records the current repository implementation fact baseline.

Future active coding work:

- Create `ACTIVE_ITERATION.md` only when coding is approved.
- Only one active iteration file may exist at a time.
- `ACTIVE_ITERATION.md` must define the current coding task, implementation scope, files allowed to be created, and validation requirements.
- After an iteration completes, conclusions should be merged back into formal documents and the working file should be removed or archived according to human instruction.

Do not use working files to bypass the authority of 00-05 plus `architecture/ADR_LOG.md`.
